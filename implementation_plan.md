# Implementation Plan: Student Management System

A prototype Student Management System for a university department, featuring student registration, course enrollment, grade submission, seed data loading, table features (searching, sorting, pagination), URL state persistence, and customized student ID validation.

## Technical Stack
- **Framework**: Vite + React + TypeScript
- **Styling**: Vanilla CSS (Custom tokens, sleek dark/light mode toggle, premium glassmorphism, responsive grid)
- **Form Validation**: `react-hook-form` + `zod` for robust schema-based validation
- **State Management**: React Context API for application-wide memory (students, courses, enrollments, grades) synced with `localStorage`
- **Routing & URL State**: `react-router-dom` for multi-page dashboard views and search parameter state sync

### Component Specifications

#### 1. Core State & Data Persistence
- **Seed Data**: Initialize the app with the provided student, course, enrollment, and grade data.
- **LocalStorage Sync**: AppContext will sync changes to `students`, `enrollments`, and `grades` keys in `localStorage`. Courses are read-only from the seed data.
- **Context API**: Provide actions like `registerStudent`, `enrollStudent`, and `submitGrade`.

#### 2. Student ID Custom Validator (`validators.ts`)
- Validation regex: `/^26[12]\d{4}$/`
- Breakdown logic check:
  - Digits 1-2: Must be `26` (for current year 2026).
  - Digit 3: Must be `1` (Morning shift) or `2` (Day shift).
  - Digits 4-7: Must be a 4-digit serial number.
- Expose a function `validateStudentId(id: string): { isValid: boolean; error?: string; details?: { year: string; shift: string; serial: string } }`.

#### 3. Forms & Schema Validation (`zod`)
- **Student Form**:
  - `id`: Must pass the Student ID validation check.
  - `name`: String, minimum 2 characters.
  - `email`: Valid email string, ends in `.edu` or general.
  - `program`: Select dropdown (`CSE`, `EEE`, `BBA`, etc.).
  - `year`: Numeric select (`1`, `2`, `3`, `4`).
- **Enrollment Form**:
  - `id`: Autogenerated (starts with `E`, e.g., `E003`) or input field with duplicate check.
  - `studentId`: Select dropdown containing registered student IDs.
  - `courseId`: Select dropdown containing available course IDs.
  - `date`: Date field, cannot be in the future.
- **Grade Form**:
  - `id`: Autogenerated (starts with `G`, e.g., `G002`).
  - `studentId`: Select dropdown containing registered student IDs.
  - `courseId`: Select dropdown containing course IDs the student is enrolled in (dynamically filtered!).
  - `grade`: Number between 0 and 100.
  - `semester`: Dropdown or input (e.g., `Spring 2026`).

#### 4. Shared Table Component
A fully featured tables UI matching the bonus criteria:
- **Search & Filter**: Real-time filtering by name/ID/program.
- **Sorting**: Clickable column headers (with indicators for `asc` and `desc`).
- **Pagination**: Configurable page sizes (e.g., 5, 10 records per page) with page navigations.
- **URL State Persistence**:
  - Synced using `react-router-dom`'s `useSearchParams` hook.
  - Example URL state structure: `?page=1&search=ayesha&sort=name&order=asc&program=CSE`.

#### 5. Premium Styling and Animation (`index.css`)
- Custom CSS variables for light/dark theme.
- CSS glassmorphism cards (`backdrop-filter: blur(12px)`).
- Micro-animations for page routing transitions, form submission, and table sort actions.
- Full responsive styling matching mobile/tablet/desktop.
